## Issues by Priority

### HIGH Priority

#### 1. Magic Strings - Extract as Constants

**From:** Code Reviewer + TypeScript Reviewer
**Location:** `.opencode/plugins/opencode-hooks.ts:260-265`

**Current:**

```typescript
const subagentType =
  input.tool === 'task' && typeof input.args.subagent_type === 'string'
    ? input.args.subagent_type
    : '';

const showToast = input.tool === 'task' && !!subagentType;
```

**Fix:**

```typescript
const TASK_TOOL_NAME = 'task';
const SUBAGENT_TYPE_ARG = 'subagent_type';

const isTaskTool = input.tool === TASK_TOOL_NAME;
const subagentType =
  isTaskTool && typeof input.args[SUBAGENT_TYPE_ARG] === 'string'
    ? input.args[SUBAGENT_TYPE_ARG]
    : '';
const showToast = isTaskTool && !!subagentType;
```

**Why:** If tool name changes, code breaks silently.

#### 2. sessionID Fallback Inconsistencies

**From:** TypeScript Reviewer
**Location:** `opencode-hooks.ts:271, 313, 340`

**Issue:** Some hooks use `input.sessionID ?? DEFAULT_SESSION_ID`, others use `input.sessionID` directly (could be undefined).

**Affected lines:**

- Line 271: `tool.execute.after` - `sessionId: input.sessionID` ❌
- Line 313: `chat.message` - `sessionId: input.sessionID` ❌
- Line 340: `chat.params` - `sessionId: input.sessionID` ❌

**Fix:** Add fallback consistently:

```typescript
sessionId: input.sessionID ?? DEFAULT_SESSION_ID,
```
